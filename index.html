Lazy Good Maybe

Someone once said that good programmers are naturally lazy. Well, I think I qualify after wrestling with the seo tool Enseomble. It turns out it was totally unnecessary. After re-coding the tool to use SharedCount and asking a friend to give it try, their response was that they currently do all of that with a product called Blockspring as a plug-in for Excel. So I researched this a bit more and found that Blockspring also works as an add-on in Google Spreadsheets. And it is really easy to use. The exact same social counts are displayed as before but without any coding. It is just a matter of using the correct cell formulas, or blocks as blockspring calls them. But wait, it gets cooler. One can code your own blocks -- i.e. custom functions for use with either Excel or Google Spreadsheets -- and in your choice of language: Python, Ruby, R, PHP, NodeJS, Javascript, and others. So whatever you can code and do in a standalone or web app you can do in blockspring. The user never has to leave the spreadsheet nor have a programmer write a custom program to gather data.
Blocks/formulas have already been written for:

  • SharedCount
  • Moz
  • SEMrush
  • HTML content extractors
  • Keyword analysis: mostly SEMrush and Alchemy API

SEO

Personally I do not work in marketing, seo, site promoting, or whatever it's called, I write software. But some of my family and friends do work in marketing and seo. They have tried on several occassions to code, in python, their own tools to help with gathering data. Sometimes with success but most often they contact me to improve or re-code their tools. So over the years I have picked up quite a bit of knowledge regarding marketing and seo -- well, I don't have any secret knowledge, skills, or weapons. But it has been interesting.
Here is a list of the tools commonly used:

  • Screaming Frog -- free for 500 URI crawls
  • SEO PowerSuite
  • Excel -- of course, who doesn't use this for everything
So with all of the above in mind, I do not think that Enseomble should reinvent software which already exists and does a great job. Also, it is important to remember that all of the data gathered is useless without someone organizing and analyzing it. So this type of work really boils down to people using spreadsheets in an intelligent way to offer their clients a beneficial service.
Here is a general video of someone doing this.

Enseomble should NOT:
  • crawl/spider web sites -- Screaming Frog does this and is free
  • scrape html content:
    title, meta description, meta keywords(now ignored by google anyways), headings, etc. Screaming Frog does all of this and it is free.

Enseomble should:
  • gather social media counts as best it can using SharedCount
  • assist with keywords and search rankings (SERPs) -- maybe ?
  • create a xlsx file for use with Excel or Google Spreadsheets
  • be as fast as possible given the expected throttling by api providers
  • be simple to use
  • run anywhere with no installation or dependencies

You might be thinking well I want something that does everything all at once and in a single software package. That sounds great, maybe someday, but for now I think the unifying piece is the spreadsheet. This allows someone doing seo work to copy, cut, paste, import, and export data as they see fit. And if the tools they need to gather that data run anywhere then they are free to focus on using spreadsheets for analysis.

While it is all well and good that things like pipulate aim to both teach python (albeit poorly) and be an seo tool, but these two things are at cross purposes. The situation has yielded a tool that is cumbersome to use and difficult to learn anything from that is valuable to anyone.
